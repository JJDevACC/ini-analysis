Package Architecture

File Organization Strategy

The R/ folder is organized by FUNCTION rather than by PIPELINE

R/
|-- pipeline.R             Pipeline entry points (Steps 1, 2, 3)
|-- io.R                   All external I/O (GIS, SCADA, PI Web API, files)
|-- validate.R             Tag classification and pump validation logic
|-- guess.R                Tag name inference from address data
|-- af_format.R            AF CSV row helper

File Details

pipeline.R

The three pipeline entry points:

Function Step Input Output
validate_gis() 1 GIS + PI Web API data/pi_tag_master_list.csv
build_af() 2 master list + SCADA data/pi_af_import.csv
export_af() 3 AF import data xlsx/pi_af_loader.csv

io.R

All functions that reach outside the R session:

Function What It Accesses
safe_get_webid() PI Web API (via get_webid())
get_ls_from_gis() GIS system (via load_gis_ps())
get_ls_by_plant() File: data/ls_by_plant.RData
get_dia_by_ls() File: data/dia_by_ls.RData
get_scada_history() PI Web API (via piwebapi::get_sampled_multi())

validate.R

Tag classification and pump validation logic:

Function Purpose
classify_tags() Split tags into 9 quality categories
print_tag_summary() Display category tree with counts
check_tags() Classify + return rows needing correction
check_all_pump_tags() Validate 16 pump tag combos per station
check_multiple_pumps_or_vfd_control() Detect multi-pump and VFD stations from SCADA history

guess.R

Tag name inference:

Function Purpose
guess_tag_name() Generate and validate address-based tag candidates
guess_tag_name_from_df() Apply guessing to all flagged assets

af_format.R

Function Purpose
af_row() Build a single row in PI AF bulk import CSV format
