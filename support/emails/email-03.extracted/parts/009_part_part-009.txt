Craig:

Nice to speak with you earlier today.  Attached is the format I used for the last import.  Note that the file must be a .csv file.

Leave the first three rows alone for an existing location.  You will notice the Location Name is in Cell A1.

Data starts on Row 4.  Column A is the date/time.  Column B is QFINAL.  Column C is QCONTINUITY.  Column D = QUANTITY.

Yes, all three data entities are the exact same number.

Convert your GPM units to MGD units.

Note:  the date/time stamp format is CRITICAL.  If this format is not correct, the import will not work.  The correct format is MM/DD/YYYY hh:mm:ss AM/PM

Once you have the .csv file formatted for import, you are ready to go to PRISM.  Follow these steps:

  1.  Go to the black toolbar in PRISM and go to the VAULT.
  2.  Once in the VAULT, look just below the bottom right of the black toolbar, and you will see UPLOAD.  Select UPLOAD.
  3.  From the list, select “Data”.
  4.  An UPLOAD DATA Dialog box will pop.  Navigate to and select the import file.
  5.  If the Location already exists in PRISM (it does for these two pump stations), and if the .csv file is formatted correctly, you will see this dialog box display:


  6.  Select IMPORT and the data should be pulled into PRISM.
  7.  Now you can go back to SLiiCER and either create a new Study or change the end date on an existing Study.

Let me know if you have any questions, or if you run into any problems.
